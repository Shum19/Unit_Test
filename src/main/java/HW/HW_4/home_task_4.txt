1. Почему использование тестовых заглушек может быть полезным при написании модульных тестов?;
Тестовые заглушки используют для упрощения тестирования: они передаются тестируемой системе вместо реальных зависимостей, настройка и сопровождение которых могут быть сопряжены с определёнными сложностями. Используя заглушки, мы получаем тест, который проверяет тестируемую систему без зависимостей, мешающих его работе. Использование тестовых заглушек снижает сложность тестов и позволяет проверять код независимо от остальной системы.;

2. Какой тип тестовой заглушки следует использовать, если вам нужно проверить, что метод был вызван с определенными аргументами?;
Stub так как возращает заданное значение для теста;

3. Какой тип тестовой заглушки следует использовать, если вам просто нужно вернуть определенное значение или исключение в ответ на вызов метода?;
Mock - возвращает нашему тестируемому методу определенный, строго заданный результат, обеспечивая таким образом предсказуемость результатов работы тестируемого модуля.;

4. Какой тип тестовой заглушки вы бы использовали для имитации взаимодействия с внешним API или базой данных?;
Fake - заменяет тяжеловесные внешние зависимые объекты легковесной реализацией.;
